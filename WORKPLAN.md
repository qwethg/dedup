# 📁 文件去重工具 — 工作计划 (v2)

> **项目名**：FileDedup — 微信/企业微信文件去重清理工具
> **目标**：一个本地运行的现代化文件去重工具，支持自然语言 AI 指令，打包为 .exe 双击即用。
> **设计原则**：简洁、现代、保持克制。不过度设计，每个功能都有明确用途。

---

## 一、技术架构

```
┌─────────────────────────────────────────┐
│              前端 (HTML/CSS/JS)           │
│   单文件 index.html，内嵌全部样式和脚本     │
├─────────────────────────────────────────┤
│              后端 (Python Flask)          │
│   本地 HTTP 服务 (localhost:18739)         │
├──────────────┬───────────────┬──────────┤
│  扫描引擎     │  操作引擎      │  AI 引擎  │
│  (文件遍历    │  (删除/移动/   │ (OpenAI  │
│   + 分组)     │   重命名)      │  兼容API) │
└──────────────┴───────────────┴──────────┘
```

- **后端**：Python + Flask，单文件 `app.py`
- **前端**：单文件 `index.html`，暗色主题，无外部依赖
- **配置**：`config.json` 持久化文件夹列表、保护列表、API Key
- **打包**：PyInstaller → 单个 `FileDedup.exe`

## 二、功能模块详解

### 模块 1：文件夹管理
- [ ] 配置持久化（config.json）
- [ ] 添加文件夹（路径 + 标签）
- [ ] 移除文件夹
- [ ] **保护文件夹**：标记为"保护"的文件夹，其中的文件不会被自动勾选
- [ ] 预设路径（微信/企业微信常用路径一键添加）
- [ ] 展示每个文件夹的文件数、总大小、保护状态

### 模块 2：文件扫描
- [ ] 递归遍历所有配置文件夹
- [ ] 文件类型过滤（默认工作文档，可切"全部"）
- [ ] 最小文件大小过滤（默认 >1KB）
- [ ] 扫描进度展示（进度条 + 当前路径）
- [ ] 扫描结果统计

### 模块 3：重复判定（三模式）
- [ ] **精确模式**（默认）：文件名 + 文件大小完全相同
- [ ] **内容模式**（可选）：MD5 哈希相同，发现不同名但内容相同的文件
- [ ] **相似模式**（可选）：文件名相同（忽略大小写），大小可能不同 → 标注"⚠️ 可能不同版本"
- [ ] 模式切换（简洁的 Tab 切换，不用下拉）

### 模块 4：前端展示

**设计风格**：简洁、现代、克制。暗色主题但不花哨。

- [ ] 统计概览卡片（4个：总文件/重复组/重复文件数/可节省空间）
- [ ] 搜索栏（文件名实时搜索）
- [ ] 扩展名筛选 chips
- [ ] **排序选项**（4种）：
  - 按可节省空间降序（默认）
  - 按文件数量降序
  - 按文件名排序
  - 按修改时间排序
- [ ] 分组卡片展示
  - 组头：文件名、扩展名 badge、份数、单份大小、可节省
  - 组内文件：勾选框、文件名、完整路径、来源标签、大小、修改时间
  - 最早文件标注"建议保留"
  - 相似模式组标注"⚠️ 可能不同版本"
  - **保护文件夹的文件标🔒图标**，仍可手动勾选但有提示
- [ ] **底部固定浮出操作栏**（勾选后浮现）

### 模块 5：操作动作（三种）

不只是删除，支持三种操作：

#### 5.1 删除
- [ ] 勾选 → 确认弹窗 → 移至回收站
- [ ] 删除后实时更新界面
- [ ] 删除日志（delete_log.txt）

#### 5.2 移动
- [ ] 弹窗选择目标文件夹
- [ ] 移动文件到指定文件夹
- [ ] **自动加后缀防冲突**：如 `file.pdf` → `file (1).pdf`
- [ ] 移动日志

#### 5.3 重命名
- [ ] 不单独做重命名面板
- [ ] 仅在移动时自动加后缀 `(1)` `(2)` 防止文件名冲突
- [ ] 如果目标位置已存在同名文件，自动递增后缀

### 模块 6：快捷规则按钮

不需要 AI 就能用的预设规则，一排简洁的按钮：

- [ ] **每组保留最新**（按修改时间，保留最近的）
- [ ] **每组保留最旧**（按修改时间，保留最早的）
- [ ] **每组保留指定文件夹的**（下拉框选文件夹标签 → 该标签的保留，其他勾选）
- [ ] 按钮点击后自动勾选，用户仍可再手动调整
- [ ] 保护文件夹的文件不会被自动勾选

### 模块 7：AI 自然语言指令

**这是我们的杀手锏功能**，市面工具没有的。

- [ ] 输入框："用自然语言描述你的规则..."
- [ ] 后端调用 OpenAI 兼容 API
  - 发送：自然语言指令 + 当前重复组数据摘要（文件名、路径、来源、大小、时间）
  - 返回：结构化 JSON 勾选指令
- [ ] 前端展示"AI 理解的规则"供审核
- [ ] 自动勾选后仍需用户确认
- [ ] 保护文件夹的文件不会被 AI 勾选
- [ ] API Key 配置入口（设置面板）

**示例指令**：
- "保留企业微信的，删其他的"
- "每份只保留最新的"
- "所有大于50MB的保留路径最短的"
- "2024年的PDF重复文件全部删除"
- "微信的优先保留"

**AI 返回格式**：
```json
{
  "understanding": "保留企业微信来源的文件，勾选其他来源的重复文件",
  "rules": [
    {"type": "keep_source", "source": "企业微信"}
  ],
  "selections": ["file_id_1", "file_id_2"]
}
```

### 模块 8：设置
- [ ] API Key 输入（OpenAI 兼容）
- [ ] API Base URL（默认 https://api.openai.com/v1）
- [ ] 模型选择（默认 gpt-4o-mini）
- [ ] 默认扫描模式
- [ ] 默认文件类型过滤

### 模块 9：导入/导出
- [ ] 导出扫描结果（JSON）
- [ ] 导出操作日志（TXT）
- [ ] 导出勾选列表（CSV）

## 六、用户流程

```
1. 双击 FileDedup.exe → 浏览器自动打开
2. 首次：添加文件夹（或点预设）→ 可标记保护
3. 点击"开始扫描"
4. 查看重复结果（切换精确/内容/相似模式）
5. 使用快捷按钮 / AI 指令 / 手动勾选
6. 选择操作：删除（回收站）/ 移动（选目标文件夹）
7. 确认执行
8. 查看日志
```

## 八、边界与安全

- ❌ 绝不自动删除，所有操作必须用户确认
- ✅ 删除 = 移至回收站，可恢复
- ✅ 移动 = 目标位置已存在则自动加后缀
- ✅ 路径安全检查（只允许操作配置文件夹下的文件）
- ✅ 保护文件夹的文件不被自动勾选（快捷规则 + AI 均不勾选）
- ✅ AI 指令只做勾选，不直接操作
- ✅ API Key 存在本地 config.json
- ✅ 所有操作有日志记录

| 功能 | 市面工具 | 我们 |

| 保护文件夹 | 部分有 | ✅ 有 |
| 打包 exe 双击即用 | 部分需安装 | ✅ 零安装 |
