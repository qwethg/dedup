# FileDedup

一个本地运行的文件去重与整理工具，专门用来清理微信 / 企业微信多年堆积的接收文件。

单文件 exe，双击即用。纯本地，免登录，文件不出本机。

---

## 为什么做这个

微信、企业微信用了几年，接收的文件全堆在一个深藏的 `FileStorage` / `WXWork` 目录里。同名文件一堆，`合同(1).pdf`、`合同(2).pdf`、`合同(2)(1).pdf`……不敢乱删，怕哪天要找的那份就在里面。

试过 dupeGuru、CCleaner 这类通用去重工具。它们能找到重复文件，但它们不懂一件事——这个文件是从微信来的，还是从企微来的。而恰恰这才是清理时最该保留的依据：工作文件一般在企微，私人文件一般在微信，删哪边留哪边，答案是不一样的。

所以做了 FileDedup。

## 它能做什么

**一键发现微信 / 企微文件夹**
不用自己去找那个藏了七八层目录的 FileStorage，启动后自动扫描常见路径，列出来让你确认。

**每个文件标注来源**
微信、企业微信、自定义文件夹，每个文件都有自己的标签。扫出来的重复组里，哪份来自哪里，一眼看清。

**用中文说规则，AI 帮你勾选**
直接打字："保留企业微信的，删其他的"、"每组留最新的那份"、"保留路径最短的"。AI 把这句话转成勾选结果，你检查一遍，再决定动不动手。
也支持不用 AI 的快捷规则：保留最新 / 最旧 / 指定来源，点一下就行。

**文件整理**
扫描结果按关键词自动分类——合同、图纸、会议纪要、报告……分不好的交给 AI 按语义归类（只发文件名，不发路径）。分完可以按类别归档到指定目录。

**删除只进回收站**
这是底线。所有删除操作都走系统回收站，可恢复。没有任何"永久删除"按钮，也绝不会在你没确认的情况下动任何文件。

**纯本地**
不登录，不上传，没有遥测。文件内容不出本机。AI 功能可选——你可以不接任何 API，只用快捷规则；也可以接本地模型（Ollama / LM Studio），完全离线。

## 快速开始

### 方式一：下载 exe（推荐给大多数人）

到 [Releases](../../releases) 下载 `FileDedup.exe`，双击运行，浏览器会自动打开操作界面。

Windows 平台，无需安装 Python 或任何依赖。

### 方式二：源码运行

```bash
git clone https://github.com/qwethg/dedup.git
cd dedup
pip install -r requirements.txt
python app.py
```

依赖很少：Flask、send2trash，没了。

### 自己打包 exe

```bash
python build.py
```

产物在 `dist/FileDedup.exe`。

## AI 功能配置（可选）

设置面板里填三项：API Base、API Key、模型名。任何 OpenAI 兼容接口都行。

- 云端模型：默认示例是 Moonshot 的 `kimi-k2.6`，换别家改一下地址和模型名即可
- 本地模型：Ollama / LM Studio 起个服务，填本地地址，一分钱不花，数据不出机器
- 隐私开关：默认关闭路径发送，AI 只能看到文件名，看不到文件在哪个目录

每次 AI 操作前，按钮悬停会显示预计的 token 消耗和成本。AI 给的勾选结果会经过服务端强制校验——保护文件夹里的文件不勾，每组保底留一个。AI 说错了也不会出事。

## 安全设计（不可妥协的几条）

1. 绝不自动删除。一切操作需要你确认，删除 = 回收站
2. 保护文件夹：被保护的文件不会被任何自动规则勾选
3. 只允许操作你配置过的文件夹，其他路径一律拒绝
4. 本地接口有随机令牌鉴权，本机其他程序冒用不了
5. API Key 只存在本地 `config.json`，不回显，不上传
6. 删除、移动、AI 操作全部写入本地日志，可查

第一次打开会要求确认一份风险声明。麻烦，但该有。

## 技术栈

- 后端：Python + Flask，单文件，AI 调用用 urllib 直连，不依赖任何 SDK
- 前端：原生 HTML / CSS / JS，零框架
- 打包：PyInstaller 单文件 exe

刻意不用前端框架和 AI SDK——东西越少，打包越省心，能跑的年头也越长。
