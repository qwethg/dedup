# 参与贡献

先谢谢你有这个念头。

这是一个从个人痛点长出来的小工具，代码结构刻意保持简单——后端一个 `app.py`，前端一个 `app.js`，没有框架，没有构建步骤。你不用先读一堆文档才能改东西。

## 提 Issue

- **bug**：说清楚操作系统、怎么触发的、期望什么、实际发生了什么。能附上报错截图更好
- **功能建议**：先说说你遇到的场景，再说你想要什么。场景比方案重要
- **吐槽**：也行，尤其想听清理微信文件时最头疼的事

## 提 PR

几件需要遵守的事：

1. **安全语义不能动**。保护文件夹不勾选、每组保底留一个、删除只进回收站、只操作配置内的路径——这几条是底线，任何改动不能削弱它们
2. 改完跑一遍测试：`python test_logic.py` 和 `python test_ai_rules.py`
3. 不引入新的重型依赖。这个项目能有今天这个体积，靠的就是克制
4. 前端继续保持零框架，后端继续保持单文件。如果你觉得这个约束挡路了，开个 Issue 聊聊再说

## 一个提醒

`config.json` 里有你的 API Key，它已经在 `.gitignore` 里。提交前 `git status` 看一眼，别把它推上去。
